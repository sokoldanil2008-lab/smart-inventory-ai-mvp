# SmartInventory AI 

**SmartInventory AI** — це інтелектуальна система для автоматизації складського обліку на малих підприємствах. Ми замінюємо паперову рутину та помилки ручного введення миттєвою оцифровкою документів за допомогою штучного інтелекту.

---

##  1. Пітч (Pitch)
Ручний облік товарів — це повільно та ризиковано. **SmartInventory AI** вирішує проблему критичних помилок та великих часових витрат. Продукт дозволяє комірникам миттєво оцифровувати надходження товарів через фотофіксацію, забезпечуючи власникам бізнесу актуальну картину залишків у реальному часі.

##  2. Ядро цінності (MVP)
На етапі MVP ми фокусуємося на одному критичному сценарії: **автоматичне розпізнавання накладної та оновлення реєстру**.
* **Дія:** Фотофіксація документа камерою смартфона.
* **Процес:** ШІ розпізнає текст та створює запис у базі даних.
* **Результат:** Користувач підтверджує коректність запису у загальному списку.

##  3. Карта стейкхолдерів
Проєкт об'єднує інтереси трьох ключових груп:
1.  **Комірники та оператори складу:** Головні користувачі, для яких спрощується щоденна робота.
2.  **Власники бізнесу (Замовники):** Оцінюють ефективність інвестицій та швидкість отримання даних.
3.  **Команда розробників:** Забезпечує стабільність алгоритмів ШІ в умовах обмежених ресурсів.



##  4. Мета першої ітерації
Створити робочий прототип системи, який дозволяє успішно пройти повний цикл: **від сканування документа до появи верифікованого запису в цифровій базі залишків.**

##  5. Ознаки успіху (Definition of Success)
Ми вважаємо MVP успішним, якщо виконуються наступні умови:
* **Автономність:** Новий користувач проходить шлях «сканування — збереження — перевірка» без додаткових інструкцій.
* **Точність:** Успішне розпізнавання стандартних друкованих накладних у **90% випадків**.
* **Швидкість:** Виконання сценарію займає **не більше 10 секунд**, що значно швидше за ручне введення.

##  Технологічний стек (планований)
* **Frontend:** React Native / Flutter (для мобільного доступу)
* **Backend:** Python (FastAPI)
* **AI/ML:** OCR (Tesseract / Google Vision API)
* **Database:** PostgreSQL / SQLite
