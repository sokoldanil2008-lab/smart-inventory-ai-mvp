# Критерії приймання MVP 

Цей документ описує вимоги до ключових функцій SmartInventory AI, які мають бути виконані для успішного релізу MVP.

---

##  1. Модуль розпізнавання 
**Сценарій:** Користувач завантажує фото стелажа для автоматичного підрахунку.

- [ ] **Точність:** Система ідентифікує мінімум 90% об'єктів на зображенні при нормальному освітленні.
- [ ] **Швидкість:** Обробка одного зображення розміром до 5МБ триває не більше 5 секунд.
- [ ] **Обробка помилок:** Якщо зображення розмите або об'єкти не розпізнані, система видає повідомлення: "Не вдалося розпізнати об'єкти, спробуйте ще раз".
- [ ] **Формати:** Система приймає файли форматів `.jpg`, `.jpeg`, `.png`.

---

##  2. Модуль прогнозування залишків 
**Сценарій:** Менеджер переглядає звіт про те, коли закінчиться товар.

- [ ] **Коректність розрахунків:** Дата вичерпання товару розраховується на основі середнього споживання за останні 7 днів.
- [ ] **Візуалізація:** Товари, яких залишилося менше ніж на 3 дні, позначаються статусом `CRITICAL` (червоний колір).
- [ ] **Дані:** Прогноз відображається для кожного SKU, що є в базі даних.

---

##  3. API та інтеграція 
**Сценарій:** Взаємодія фронтенд-частини з сервером.

- [ ] **Документація:** Всі ендпоінти (`/scan`, `/forecast`, `/items`) описані в Swagger (доступно за адресою `/docs`).
- [ ] **Стабільність:** Сервер повертає статус `200 OK` при успішних запитах та коректні коди помилок (`400`, `404`, `500`) при збоях.
- [ ] **Безпека:** API приймає запити лише з дозволених доменів (CORS налаштовано).

---

##  4. Інтерфейс користувача 
**Сценарій:** Користувач відкриває головну сторінку системи.

- [ ] **Адаптивність:** Веб-інтерфейс коректно відображається як на десктопі, так і на екрані смартфона.
- [ ] **Відображення даних:** Таблиця залишків оновлюється миттєво після успішного сканування.
- [ ] **Простота:** Користувач може провести повний цикл (завантаження фото -> отримання результату) за 3 кліки.

---

##  5. Розгортання 
- [ ] **Docker:** Проект містить `Dockerfile`, і його можна запустити однією командою `docker-compose up`.
- [ ] **Логи:** Система веде лог запитів для можливості відстеження помилок.
